# Contribution Guide

Welcome to the Warp Session Suite! We appreciate your interest in contributing to this project. Whether you're here to file a bug report, suggesting new features, or contributing code, your efforts are greatly appreciated!

## Code of Conduct

This project adheres to a Contributor Covenant [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report any unacceptable behavior to [maintainers@example.com].

## How Can I Contribute?

### Bug Reports

To file a bug report, use the GitHub Issues page. Ensure your report includes concise title and a clear description, with as much relevant information as possible.

### Suggesting Enhancements

Feature requests can also be filed as GitHub Issues. Explain your desired enhancement, why it would be useful, and any suggested implementation details.

### Pull Requests

1. Fork the repository.
2. Create a new branch: `git checkout -b feature/your-feature`
3. Make your changes and commit: `git commit -am 'Add your feature'`
4. Push to your branch: `git push origin feature/your-feature`
5. Create a Pull Request.

## Style Guide

- Follow standard linting and formatting rules using tools stated in `.tool-versions`.
- Ensure code is tested appropriately.
